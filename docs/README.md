# WorkSphere Documentation Suite

Welcome to the comprehensive documentation suite for **WorkSphere**, an enterprise-grade SaaS platform for digital workplace collaboration, task management, sprint scheduling, and staff leave approvals.

These documents serve as the single source of truth for human engineers and AI coding assistants alike. They describe the system's architecture, patterns, permissions, and coding guidelines in detail.

## Documentation Map

To understand or contribute to the system, refer to the following guides:

| Document | Purpose | Audience |
| :--- | :--- | :--- |
| 📁 **[Project Structure](PROJECT_STRUCTURE.md)** | Explains the folder hierarchy, purpose of directories, placement rules, and cross-folder relations. | All Developers |
| 🏛️ **[System Architecture](ARCHITECTURE.md)** | Covers frontend/backend stacks, request-response flows, validation layers, authentication protocols, and database separation. | System Architects / New Developers |
| 🛡️ **[Enterprise Role System](ROLE_SYSTEM.md)** | Maps all client routes, CRUD privileges, widgets, and API controls for Admin, Manager, Team Lead, and Employee roles. | Product Owners / QA Engineers |
| 💾 **[Database Schemas](DATABASE_SCHEMA.md)** | Documents every Mongoose schema model, data validations, indexing rules, hooks, and collections links. | Database Admins / Backend Developers |
| 🔌 **[API Reference Guide](API_REFERENCE.md)** | Details all Express route mappings, request parameters, JSON bodies, authentication status codes, and payload responses. | Backend / Integration Developers |
| 🔄 **[Developer Workflows](WORKFLOW.md)** | Outlines checklists and step-by-step methods for creating pages, adding features, defining permissions, or scaling modules. | Human Software Engineers |
| 🤖 **[AI Coding Agent Guide](AI_AGENT_GUIDE.md)** | Sets explicit syntax, state conventions, reuse requirements, and architectural rules for AI tools (Windsurf, Claude Code, Cursor). | AI Coding Companions / Prompt Engineers |
| 🤝 **[Contributing Standard](CONTRIBUTING.md)** | Guides repository onboarding, database seeding, branch names, pull request rules, and testing lists. | Teammates & External Contributors |
| 🗺️ **[Codebase Reference Map](CODEBASE_MAP.md)** | Provides a quick lookup guide linking client pages to controllers, services, schemas, and API routers. | Debugging Developers |

## Workspace Quick Start

If you are a new developer or AI assistant, follow this suggested reading order:
1. **[Architecture](ARCHITECTURE.md)** to grasp the flow of request-response blocks.
2. **[Project Structure](PROJECT_STRUCTURE.md)** to locate files.
3. **[AI Agent Guide](AI_AGENT_GUIDE.md)** or **[Developer Workflows](WORKFLOW.md)** to plan edits or new code.
